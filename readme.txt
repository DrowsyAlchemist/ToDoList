	ToDoList

Фичи:

- создать/изменить/удалить задачу
- открыть список всех задач/задачи на сегодня/задачи на произвольный день
- уведомление по времени задачи (если задано)

- создание пользователя, авторизация, редактирование профиля

- MVC, логгирование, кэширование, обработка исключений, Health Check

Интерфейс: 

	- окно ошибок
	- окно авторизации (перекидывает в окно задач)
	- меню профиля (отображаемое имя, изменить пароль, поставить аватар)
	- окно задач (список задач, селектор задач(все/на сегодня/да другой день), 
	  кнопка добавления задачи (перекидывает в меню добавления))
	- задачи: название, время, приоритет, статус. По нажатию перекидывает в меню редактирования задачи
	- меню добавления/редактирования задачи: простая форма со всеми данными модели
	
Модель: 

	- пользователь: id, email, password, name, avatar
						--(login data)-
	- задача: 
		название, 
		описание, 
		дата и время, 
		приоритет (low, middle,  high),
		статус (active, pending, cancelled, overdue)

С чего начать: Логгирование и обработка ошибок -> инфраструктура MVC -> представления 
																		(можно использовать всякие хелперы или просто html)


Баги:

	- проблема с датой и временем
	- при редактировании слетает статус и приоритет (проблема с селектором)
	- нужно узнать путь, ОТКУДА пользователь перешел на страницу

		- сделать кнопку для пререхода на страницу админа (отображается только админам)
		- при нажатии на пользователя отображается ЕГО список задач
	- привязать задачи к конкретным пользователям
	- отрефакторить LoginController
	- сделать класс Role enum-ом
